# 戦略会議：どのようなページにするか

**Aさん**：LP-OS Master スキル（CVR・変換設計）  
**Bさん**：デザインスキル・ディレクター（Awwwards 基準・ビジュアル品質）  
**テーマ**：ハートフルクリーン（遺品整理）のページ戦略

---

## 1. 会議の目的

「問い合わせ・電話・LINE を増やしたい」と「信頼され、記憶に残るブランドにしたい」の両立を、どのページ構成・ビジュアルで実現するかを決める。

---

## 2. Aさん（LP-OS）の主張

### 2-1. 前提

- **LP = ユーザーの心理変容を導く変換装置**。売り手目線を排除する。
- 離脱の大部分は **3秒以内** に起きる。FV で「誰が・何をしてくれるか」を結論として出さないと、スクロールすらされない。
- ターゲットは **潜在層**（悩みはあるが解決を諦めかけている・後回しにしている）なので、**PASONA** が有効。結論ファースト → Problem → Affinity → Solution → Offer → Narrowing → Action の順を崩すと CVR が落ちる。

### 2-2. ページに必須のもの

1. **FV の必須4要素**  
   キャッチ（ベネフィット＋数字）・メイン画像（UGC 風）・CTA（Above the Fold）・権威バッジ。4つ揃わない FV は仕様違反。

2. **低ハードル CTA**  
   「今すぐ電話」は重い。「無料で話を聞いてもらう」「状況をLINEで送るだけ」のように、**具体的かつ心理的負荷の低い**ラベルにしないとクリック率が伸びない。

3. **行動経済学トリガーの明示配置**  
   損失回避（後回しのリスク）・バンドワゴン（累計件数・★5.0）・ウィンザー（お客様の声）・リスクリバーサル（急がない・断っても可）を、セクションごとに「何のためか」を意識して置く。

4. **インタラクティブ要素**  
   CPC 高騰下では、診断クイズなどで **一貫性の原理** を働かせ、「自分はこの会社に頼もうとしている」と自己認識させる設計が CVR を底上げする。

5. **誠実トーン（2026 Mandatory）**  
   煽り・誇張は禁止。でも「伝える順序」と「CTA の言い方」はアルゴリズムで決める。

### 2-3. デザインへの注文

- **美しさは手段**。CVR が取れない美しさは、この案件ではゴールではない。
- ただし **UGC 風の写真・リアルな質感** は LP-OS も推奨している。広告感・AI 感の強いビジュアルはブランド毀損なので却下。
- **Z の法則・F の法則** で、視線の先に CTA を置く。人物の視線の先にボタンを配置する、というのはデザインと共通。

### 2-4. 懸念（B さんへの問い）

- 余白を極端に取りすぎると、**FV に CTA が入りきらない**。3秒ルールと「余白の勇気」のどちらを優先するか、ルールを決めたい。
- セクションの「リズムを変える」ために見出しを削ると、**PASONA の型が崩れる**可能性がある。どこまで型を崩してよいか、線引きが欲しい。

## 3. Bさん（デザイン D）の主張

### 3-1. 前提

- **すべての判断は「Awwwards 8.0 を取れるか」で行う**。美しさ・使いやすさ・独自性・コンテンツの 4 軸で測る。
- 「おしゃれさ」は **余白・タイポグラフィ・カラー・リアリティ** の 4 要素を同時に制御した結果。感覚ではなくルールで再現する。
- **チープとプレミアムの境界**：全部中央揃え・均等グリッド・同一パターン 3 回以上 → チープ。非対称・リズムの変化・「記憶に残る 1 点」→ プレミアム。

### 3-2. ページに必須のもの

1. **余白のルール**  
   セクション間 120〜200px。関連要素は近く、非関連は遠く。**余白と密度のコントラスト**がないと「スカスカ」か「詰め込み」のどちらかになる。

2. **タイポのジャンプ率**  
   見出し／本文は **3 倍以上**。和文は weight 500 まで。数字は欧文・tabular-nums。ここが守られていないと「安いテンプレ」に見える。

3. **色は 7 色以内・80/15/5**  
   背景はオフホワイト。テキストはソフトブラック。アクセントは **1 色に絞る**。CVR 用に CTA を補色にするのは賛成だが、それ以外で色が増えすぎると散漫になる。

4. **同一パターンの連続禁止**  
   section-label + section-heading + コンテンツ が 3 回以上続くと「テンプレート」と判定される。**1 セクションは見出しなし・数字だけ**など、意図的な破りが 1 つは必要。

5. **リアリティ（写真）**  
   現場・実写・加工されていない質感を優先。AI 生成・完璧すぎるストックは信頼を下げる。LP-OS の UGC 推奨と一致。

6. **記憶に残る 1 点**  
   コンテンツの強さ（元警察官・狩野さん）だけでは不十分。**デザインだけで「あのサイト」と言えるシンボル**（オープニングの円の再登場・独自の 1 要素など）が 1 つあると、創造性のスコアが伸びる。

### 3-3. CVR への注文

- **導線と CTA の位置** は A さんの設計に従う。ただし CTA の **形・色・マイクロコピー** は、全体のトーンと 80/15/5 を崩さない範囲でデザイン側で調整したい。
- **診断クイズ** は、見た目が「他サイトのクイズ」と同一にならないよう、色・余白・タイポでこのサイト用に設計する。

### 3-4. 懸念（A さんへの問い）

- FV に要素を詰め込みすぎると **3 秒で「忙しい」と感じて離脱** する可能性もある。必須 4 要素を **視覚的には 2 ブロックにまとめる**（左：キャッチ＋バッジ、右：画像＋CTA）など、情報量は守りつつ「余白の呼吸」を入れる設計は許容されるか。
- PASONA の **順序は変えずに、見た目だけリズムを変える**（例：Problem は引用 1 ブロック＋損失で「型」は P のまま、レイアウトは中央ではなく左寄せ）は、A さん的には OK か。

## 4. すり合わせと合意

### 4-1. ゴールの優先順位（合意）

1. **第一目標**：問い合わせ・電話・LINE のコンバージョン増（CVR）。
2. **第二目標**：信頼と「この業者なら」という記憶に残るブランド。煽らず誠実なトーン。
3. **第三目標**：業界平均を超える「きれいさ・一貫性」。テンプレート感を出さない。

→ **CVR を損なわない範囲で、デザインルール（余白・タイポ・色・リズム）を適用する** で進める。

### 4-2. FV について（合意）

- **必須 4 要素は崩さない**（A 案）。キャッチ・画像・CTA・権威バッジは必ず FV 内に収める。
- **レイアウト**：左にテキスト（キャッチ＋バッジ＋副コピー＋損失 1 行＋CTA）、右に画像。Z の法則に沿い、**人物の視線の先に CTA が来る** ように画像側の余白や配置を調整する（B 案）。
- **余白**：FV 内でも、ブロック間は 16〜24px 確保し、「詰め込み」に見えないようにする（B 案）。ただし CTA はスクロールなしで可視（A 案）。

### 4-3. セクション構成とリズム（合意）

- **結論ファースト ＋ PASONA の順序は変えない**（A 案）。Problem → Affinity → Solution → Evidence → Offer → Narrowing → Action を維持。
- **見た目のリズム**は B 案で変更してよい。例：Problem は「見出し＋6 カード」ではなく **引用 1 ブロック＋損失ブロック** にし、Affinity は **3 枚のカード** のまま見出しを左寄せにする、など。**中身の役割（P/A/S…）** は A、**レイアウト・余白・見出しの有無** は B がリード。

### 4-4. CTA とマイクロコピー（合意）

- **ラベル**は A 案。「無料で話を聞いてもらう」「状況をLINEで送るだけ」など低ハードルに統一。
- **色・サイズ・角丸**は B 案。メインカラーの補色（緑）で CTA、タップ領域 44px 以上、角丸は 6〜8px で他コンポーネントと揃える。
- **Risk Reversal** の文言は A 案。配置とフォントサイズ・余白は B 案。

### 4-5. 写真・ビジュアル（合意）

- **UGC 風・実写・現場** を優先。広告感・AI 感は両者とも NG。
- 既存の打ち合わせ写真・代表写真・現場写真をそのまま活用。B 案で **アスペクト比・余白・配置** を整える。

### 4-6. 診断クイズ（合意）

- **一貫性の原理** を狙ったインタラクティブ要素として残す（A 案）。
- **見た目**は C パターン専用の色・余白・タイポで、他サイトと被らないようにする（B 案）。ボタンやカードのスタイルはサイト全体のトークンに合わせる。

### 4-7. 「記憶に残る 1 点」（合意）

- **コンテンツ**（元警察官・狩野さん・追加料金ゼロ）は A が設計。**ビジュアルの 1 点**（オープニングの円の再登場・FV や final-cta の 1 要素など）は B が提案し、A は「CVR を妨げない」条件で了承する。

## 6. ページ別の戦略（A・B・C の位置づけ）

- **A パターン（index.html）**  
  **B さんリード**の「感情導線＋静寂・信頼・デザイン品質」。A さんの要素（感情導線・信頼 4 要素・CTA 具体化）はすでに反映済み。今後は B の余白・リズム・記憶に残る 1 点をさらに強化。

- **B パターン（index-b.html）**  
  **B さんリード**の「余白とことば・画像スロット・自由構成」。A さんの観点では「CTA と権威は必ず入れる」「セクションの役割は読めるようにする」を満たす範囲で B の表現を優先。

- **C パターン（index-c.html）**  
  **A さんリード**の「LP-OS 完全準拠・CVR 特化」。構成・コピー・CTA は A。B さんは **色・余白・タイポ・コンポーネントの一貫性** を C 用トークンで整え、テンプレート感を出さない。

→ **どのページも「CVR を損なわない範囲でデザインルールを適用」「デザインで CVR を妨げない」** を共通原則とする。

## 7. 次のアクション

1. **C パターン**：B さんが C 用の余白・タイポ・「1 セクションだけリズムを変える」をチェックし、必要なら index-c.html に反映する。
2. **A パターン**：B さんが「記憶に残る 1 点」の具体案（オープニング円の再登場など）を 1 つ提案。A さんが CVR 観点で OK 出し。
3. **共通**：写真は UGC 風・実写を維持。新規写真を足す場合は両者でトーンを確認。
