# C案（index-c.html）実装フェーズ

完全企画（A・B・C 3者合意）に基づき、C案を**どのフェーズで進めるか**を整理した進行表です。上から順に進め、各フェーズの成果物と担当を明確にします。

---

## 全体フロー

```
フェーズ1：仕様・構成の確定
    ↓
フェーズ2：コピー確定（文言ドラフト → 確定）
    ↓
    【D：辛口評論家 1回目レビュー（文言・流れ）】
    ↓
フェーズ3：デザイン・トークン適用（余白・タイポ・フォント・アニメ）
    ↓
フェーズ4：実装（HTML/CSS/JS 反映・診断クイズ・reduce-motion）
    ↓
フェーズ5：品質チェック（A/B/C ＋ D：辛口評論家 2回目＝実機レビュー）
    ↓
フェーズ6：公開準備（写真・計測・最終確認）
```

---

## フェーズ1：仕様・構成の確定

**目的**：C案の「何をどこに表示するか」を全員で揃える。

| 項目 | 内容 | 担当 | 成果物 |
|------|------|------|--------|
| PASONA 対応表 | 各セクションが P/A/S/Evidence/Offer/Narrowing/Action のどれかと一致しているか | A | セクション一覧表（既存の完全企画 §2 でほぼ済） |
| 必須4要素 | FV にキャッチ・画像・CTA・権威バッジが入っているか | A | チェック済み一覧 |
| 診断クイズ | 設問数・選択肢・結果→CTA の流れが決まっているか | A | 診断クイズ仕様（設問・分岐・結果メッセージの骨子） |
| 写真・素材 | 使用する写真（代表・現場・UGC 風）のリストと差し替え可能か | A・B | 写真一覧・スロット表 |

**終了条件**：構成と診断クイズの骨子に A が OK を出し、B・C が「この順でコピー・デザインを当てる」と了解していること。

**成果物**：`C案_フェーズ1_成果物.md`（PASONA 対応表・必須4要素チェック・診断クイズ仕様骨子・写真一覧）

## フェーズ2：コピー確定

**目的**：全セクションの文言を「完全企画 §3（セクション別コピー設計）」と C のルールに沿って確定する。

| 項目 | 内容 | 担当 | 成果物 |
|------|------|------|--------|
| FV | キャッチ・副コピー・損失1行・CTA ラベル・LINE ラベル・マイクロコピー | C | 文言シート（FV） |
| Problem | 引用（ユーザーの声）・損失ブロック1文 | C | 文言シート（Problem） |
| Affinity | 3カード分「不安→解消」の見出し＋本文 | C | 文言シート（Affinity） |
| Solution | 代表経歴・代表引用（実在の言葉・要確認） | C | 文言シート（Solution）※代表承認推奨 |
| Evidence | 数値見出し・理由3本・お客様の声2〜3件（出典明記） | C | 文言シート（Evidence） |
| 診断クイズ | 設問文・選択肢・結果メッセージ・CTA 誘導文 | C | 文言シート（診断クイズ） |
| Offer | サービス名＋1行 Benefit・料金表・注記 | C | 文言シート（Offer） |
| Narrowing | 「選ばれている方」・「ご注意」 | C | 文言シート（Narrowing） |
| Action | 見出し・Risk Reversal 箇条書き・CTA 周辺 | C | 文言シート（Action） |
| 禁止チェック | §2-9 禁止フレーズ・企業主語・FAB 未達の有無 | C | チェック済み印 |

**終了条件**：全ブロックの文言がドラフト→確定し、禁止チェックを C が完了していること。必要なら A が「構成・トリガーに合っているか」を確認。

**成果物**：`C案_フェーズ2_文言シート.md`（FV〜Action・診断クイズ Q1〜Q3・4パターン結果・禁止チェック済み）

**4人目 D（辛口評論家）**：フェーズ2 終了後、文言シートを D に渡し 1回目レビューを依頼。指摘リストは C（と必要なら A）が対応。詳細は `4人目（辛口評論家）の投入方針_3者合意.md` 参照。

## フェーズ3：デザイン・トークン適用

**前提**：D（辛口評論家）1回目指摘を回収し、文言シートに反映済み。アイコン・絵文字禁止ルールを `C案_AI制作感対策・アイコン禁止.md` で追加済み。この状態でフェーズ3に着手する。

**目的**：完全企画 §4（ビジュアル・トークン）と §5（フォント・アニメーション）を C案に反映する。

| 項目 | 内容 | 担当 | 成果物 |
|------|------|------|--------|
| **アイコン・絵文字** | **使用禁止**。CTA・バッジ・Offer 見出し・Narrowing はすべて文言のみ（`C案_AI制作感対策・アイコン禁止.md`） | B・実装 | 実装時（フェーズ4）に削除。デザイン時点でアイコンを仕様に含めない。 |
| 色 | 紺・緑 CTA・ゴールドの 80/15/5、7色以内の確認 | B | index-c の CSS 変数レビュー |
| 余白 | セクション間 120〜200px、FV 内 16〜24px | B | 余白一覧 or 修正パッチ |
| タイポ | 見出し/本文 3倍以上、和文 500 まで、数字 tabular-nums | B | フォント指定・clamp 値の見直し |
| フォント | Noto Sans JP / Noto Serif JP / **Inter** の読み込みと割り当て（§5-1） | B | link タグ追加・クラス割り当て |
| リズム | 同一パターンが3回続いていないか、1セクションだけ見出しなし or 左寄せで破る | B | レイアウトメモ |
| アニメーション | スクロール fade-in / 簡易オープニング or なし、hover 200–400ms、--c-ease（§5-2） | B | アニメーション仕様メモ |
| reduce-motion | prefers-reduced-motion でアニメ無効化の有無 | B | 実装有無の確認 |

**終了条件**：B が「C案に §4・§5 を適用した」と宣言し、フォント・余白・アニメの実装チェックリスト（§5-3）を満たしていること。

## フェーズ4：実装

**目的**：フェーズ2の文言とフェーズ3のトークンを index-c.html に反映し、診断クイズとアクセシビリティまで実装する。

| 項目 | 内容 | 担当 | 成果物 |
|------|------|------|--------|
| 文言の差し替え | 全セクションのテキストを「文言シート」で置き換え | 実装 | 更新された index-c.html |
| **アイコン・絵文字削除** | CTA・Offer・Narrowing・損失ブロック等から **すべて削除**（`C案_AI制作感対策・アイコン禁止.md`） | 実装 | 絵文字・SVG アイコンなしの index-c.html |
| 診断クイズ | 設問・選択肢・結果・CTA 誘導の JS とマークアップ | 実装 | 動作する診断クイズ |
| フォント | Inter 追加、tabular-nums の適用箇所 | 実装 | 反映済み index-c.html |
| アニメーション | スクロール用クラス（fade-in 等）、hover、必要なら簡易オープニング | 実装 | 反映済み index-c.html |
| reduce-motion | @media (prefers-reduced-motion: reduce) で無効化 | 実装 | 反映済み index-c.html |
| 写真・画像 | スロットに実際の画像パス or プレースホルダー方針 | 実装 | 画像反映 or 差し替えリスト |

**終了条件**：index-c.html がローカルで意図どおり表示・動作し、スマホ表示と reduce-motion の両方を確認していること。

## フェーズ5：品質チェック

**目的**：A・B・C の観点で最終レビューし、公開可能な水準にする。

| 観点 | チェック内容 | 担当 |
|------|--------------|------|
| **A（構成・CVR）** | FV 必須4要素、PASONA 順序、CTA 低ハードル、トリガー配置、診断→CTA の導線 | A |
| **B（デザイン）** | 余白・ジャンプ率・色数・同一パターン3回禁止・写真トーン・§5-3 実装チェックリスト・**アイコン・絵文字なし**（`C案_AI制作感対策・アイコン禁止.md`） | B |
| **C（コピー）** | 禁止フレーズ・主語・FAB・CTA 丁寧語・1セクション1メッセージ・裏付けのない数字なし | C |
| **D（辛口評論家）** | 実機で「信じない・動かない・離脱しそう」な点を指摘。指摘リスト提出。採用可否は A/B/C が判断。 | D |
| **E（AIらしさ診断）** | 実機で「AIが作ったと悟られるか」を診断。anti-ai-aesthetic 準拠の診断レポート（チェック○×＋該当箇所の理由＋「このサイトならでは」の有無）提出。対応は主に B。 | E |
| **共通** | リンク切れ、電話番号・LINE URL、表示崩れ（複数ブラウザ・幅）、読み上げ・キーボード操作 | 全員 |

**終了条件**：A・B・C がそれぞれの観点で「問題なし」または「軽微な指摘のみ（対応済み）」とすること。D の指摘は対応方針（採用／不採用＋理由）を決めて記録する。E の診断は B が主に対応し、該当箇所の修正方針を記録する。詳細は `4人目（辛口評論家）の投入方針_3者合意.md` および `5人目（AIらしさ診断）の投入方針_3者合意.md` 参照。

**フェーズ5 実施済み**：結果は `C案_フェーズ5_品質チェック結果.md` に記録。A・B・C・D・E いずれも「問題なし」または「軽微な指摘のみ（対応任意）」。総合判定は公開可能水準。

## フェーズ6：公開準備

**目的**：本番環境に出すための最終確認と計測準備。

| 項目 | 内容 | 担当 |
|------|------|------|
| 写真・権利 | 使用写真の権利・肖像権・差し替え可否の確認 | クライアント or A・B |
| 計測 | GA4・GTM・コンバージョンタグの設置方針（C案専用 URL やパラメータ） | A or 計測担当 |
| メタ・OGP | title / description / OGP の最終文言（C 案用） | C |
| リリース | デプロイ手順・公開日・A/B テストの有無 | プロジェクト |

**終了条件**：公開に必要な素材・タグ・手順が揃い、責任者が GO を出せる状態であること。

**成果物**：`C案_フェーズ6_公開準備.md`（写真・権利チェック、計測方針、メタ・OGP 最終案、リリース手順）。チェックを埋めていき、GO が出たら公開。

## 進め方の目安

- **フェーズ1**：既に完全企画でほぼ確定しているため、診断クイズの骨子と写真リストの有無を確認すれば 1 日以内で終了可能。
- **フェーズ2**：C が文言を一括でドラフトし、禁止チェックと代表引用の確認を含めても 2〜3 日想定。
- **フェーズ3**：B がトークンと §5 を index-c に当てる。既存の C 用変数があるため 1〜2 日。
- **フェーズ4**：文言反映・診断クイズ・フォント・アニメ・reduce-motion を実装。2〜3 日想定。
- **フェーズ5**：A/B/C でレビュー。指摘の反映に 1 日。
- **フェーズ6**：写真・計測・メタの最終確認。クライアント確認を含めると数日〜1 週間程度。
